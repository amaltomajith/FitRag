# FitRAG — AI Fitness & Nutrition Chatbot

A full-stack RAG-powered chatbot that gives personalised fitness and nutrition advice
grounded in a curated knowledge base of 25 foods (5 cuisines) and 25 exercises.

## Stack
- **Backend**: Python FastAPI + ChromaDB + sentence-transformers + Groq LLM
- **Frontend**: React + Vite + Tailwind CSS v4
- **Storage**: SQLite (users + messages), ChromaDB (vector store)
- **LLM**: Groq `llama-3.3-70b-versatile` via OpenAI-compatible client

## Quick Start

### 1. Backend setup

```bash
cd backend

# Install Python deps
pip install -r requirements.txt

# Set your Groq API key
$env:GROQ_API_KEY = "gsk_..."   # PowerShell
# OR: export GROQ_API_KEY="gsk_..."  # bash/zsh

# Ingest the knowledge base into ChromaDB (run once)
python ingest.py

# Start the API server
uvicorn main:app --reload --port 8000
```

### 2. Frontend setup

```bash
cd frontend
npm install
npm run dev
```

Open http://localhost:5173

## Project Structure

```
backend/
  main.py              # FastAPI app (profile, chat, history)
  ingest.py            # ChromaDB embedding script
  requirements.txt
  knowledge_base/
    foods.json         # 25 foods across 5 regions
    exercises.json     # 25 exercises with form cues
  chroma_db/           # Auto-created by ingest.py
  app.db               # Auto-created by FastAPI on startup

frontend/
  src/
    App.jsx            # Root: profile gate → chat
    ProfileForm.jsx    # Height/weight/region/goal form
    ChatWindow.jsx     # Message bubbles, typing indicator
    api.js             # fetch wrappers for /api/*
  index.css            # Tailwind v4 + custom animations
```

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/profile` | Create/update user profile → returns `user_id` |
| POST | `/api/chat` | `{user_id, message}` → RAG-grounded LLM reply |
| GET | `/api/history/{user_id}` | Full message history |
| GET | `/health` | Health check |

## Deployment

This project is set up to be deployed with **Railway** (for the persistent python backend) and **Vercel** (for the frontend).

### 1. Backend (Railway)
The backend requires a persistent environment to store SQLite (`app.db`) and ChromaDB vector store data.

1. Go to [Railway](https://railway.app) and create a new project from your GitHub repository.
2. Railway will auto-detect the configuration using the root `nixpacks.toml` and `railway.toml`.
3. In your Railway service settings under **Variables**, add the following environment variables:
   - `GROQ_API_KEY`: Your Groq API key.
   - `YOUTUBE_API_KEY`: Your YouTube Data API v3 key.
   - `ALLOWED_ORIGINS`: Comma-separated list of allowed origins, e.g., `https://your-app.vercel.app` (update this once your frontend is deployed).
4. In your Railway service under **Volumes**, add a persistent volume mounted at `/app` to ensure SQLite and ChromaDB data persists across restarts.
5. In the Railway service terminal/shell, run the database ingestion script once:
   ```bash
   cd /app && /app/.venv/bin/python backend/ingest.py
   ```

### 2. Frontend (Vercel)
1. Go to [Vercel](https://vercel.com) and import your GitHub repository.
2. Select `frontend` as the root directory of the Vercel project.
3. In **Environment Variables**, add:
   - `VITE_API_URL`: The public URL generated by your Railway backend service (e.g., `https://your-backend.up.railway.app`).
4. Click **Deploy**.
